# SELF-DIRECTED EXCELLENCE FRAMEWORK

A compact checklist and templates for continuous improvement, decision-making, and learning.
Use this as your two-week operating rhythm and living reference.

## Quick Checklist (two-week loop)

- [ ] Problem clarity: define the problem, the user, severity, why unsolved, boundaries.
- [ ] Current state: what's working, what's broken, root cause, constraints, assumptions, data.
- [ ] Threat model: top threats, easiest/highest-impact attack, data at risk, first defenses.
- [ ] Scale checks: what breaks at 10x/100x, real bottlenecks, cache/parallelize opportunities.
- [ ] Failure modes: worst-case, detection, recovery time, data loss, user impact.
- [ ] Metrics: primary metric, what's measured, measurement gaps, where pain happens.
- [ ] Hypothesize: write 1 small testable hypothesis and expected metric change.
- [ ] Experiment: run smallest experiment, measure, stop fast, record results.
- [ ] Iterate: keep what works, analyze failures, define next experiment or decision.
- [ ] Deep work: schedule focused 4-hour session for root-cause or high-leverage work.
